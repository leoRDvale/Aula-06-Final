# Aula-06-Final

1) Considere uma empresa de notificações que precisa enviar mensagens para seus clientes por diferentes canais: E-mail, SMS e Push Notification (notificação para aplicativo mobile). Crie um aplicativo que deve ser flexível o suficiente para que novos canais de notificação possam ser adicionados no futuro sem alterar o código já existente. Para isso, você deverá implementar o padrão de projeto Factory Method, garantindo que a criação dos objetos de notificação seja feita de forma desacoplada e extensível.

Obs.: Salve no PDF o Link do Repositório da aplicação 1

2) Adicione um quarto canal de notificação: WhatsAppNotificacao, sem modificar nenhuma classe já existente. Isso demonstrará na prática o Princípio do Aberto/Fechado (OCP) do SOLID, que o padrão Factory Method naturalmente favorece. 

Obs.: Salve no PDF o Link do Repositório da aplicação 2. Cada aplicação deve ficar no seu repositório.

Obs.: Cada aplicação deve ter seu UML
